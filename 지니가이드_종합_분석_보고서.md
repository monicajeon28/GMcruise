## 2. 긴급 수정 필요 (Critical)

### 🔴 문제 1: 환경 변수 누락으로 인한 기능 중단 위험

**영향 범위**: 매우 높음  
**우선순위**: 🔴 최우선

#### 발견된 문제점

1. **Gemini API 키 누락 시 AI 기능 전면 중단**
   - 위치: `lib/gemini.ts:3-7`
   ```typescript
   const GEMINI_API_KEY = process.env.GEMINI_API_KEY || process.env.GOOGLE_GENERATIVE_AI_API_KEY;
   if (!GEMINI_API_KEY) {
     throw new Error('Missing GEMINI_API_KEY');
   }
   ```
   - 영향:
     - AI 채팅 기능 완전 중단
     - 번역 기능 중단 (`app/api/chat/route.ts:68-74`)
     - 이미지 분석 기능 중단 (`app/api/vision/route.ts:10-19`)
   - 현재 상태: 에러 처리 있음 (500 에러 반환)

2. **Google Drive 인증 정보 누락 시 파일 업로드/조회 불가**
   - 위치: `lib/google-drive.ts:25-58`
   - 필수 환경 변수:
     - `GOOGLE_DRIVE_SERVICE_ACCOUNT_PRIVATE_KEY` (또는 대체 변수)
     - `GOOGLE_DRIVE_SERVICE_ACCOUNT_EMAIL` (또는 대체 변수)
   - 영향:
     - 이미지 업로드 실패
     - 여권 업로드 실패
     - 크루즈 사진 조회 실패
     - 계약서 업로드 실패
   - 현재 상태: 에러 처리 있음 (throw Error)

3. **데이터베이스 연결 문자열 누락 시 전체 시스템 중단**
   - 위치: `prisma/schema.prisma:8`
   ```prisma
   url = env("DATABASE_URL")
   ```
   - 영향: 모든 데이터베이스 작업 실패
   - 현재 상태: Prisma가 자동으로 에러 처리

#### 해결 방안

1. **환경 변수 검증 스크립트 실행**
   ```bash
   npm run env:check
   ```

2. **Vercel 환경 변수 설정 확인**
   - Vercel 대시보드 → 프로젝트 → Settings → Environment Variables
   - 필수 변수 목록 확인 (아래 "환경 변수 의존성 분석" 참조)

3. **환경 변수 누락 시 우아한 에러 처리**
   - 현재: 일부는 throw Error로 앱 중단
   - 개선: 환경 변수 누락 시 사용자 친화적 메시지 표시

### 🔴 문제 2: 관리자 로그인 인증 불일치 문제

**영향 범위**: 관리자 기능 전면 중단  
**우선순위**: 🔴 최우선

#### 발견된 문제점

1. **전화번호 정규화 불일치** (이미 문서화됨)
   - 위치: `app/api/admin/auth-check/route.ts`
   - 문제:
     - 로그인 API에서는 전화번호 정규화 (`phone.replace(/[-\s]/g, '')`)
     - 인증 확인 API에서는 정규화하지 않고 직접 비교
   - 영향: 로그인은 성공하지만 관리자 패널 접근 불가
   - 상태: ✅ 수정 완료 (문서에 따르면)

2. **Prisma 클라이언트 모듈 찾기 실패 가능성**
   - 위치: Vercel 배포 시
   - 문제: `Cannot find module '.prisma/client/default'` 오류 가능
   - 영향: DB 조회 실패로 인증 확인 불가
   - 상태: ✅ `next.config.mjs`에서 제외 설정 제거 완료 (문서에 따르면)

#### 해결 방안

1. **관리자 계정 확인 스크립트 실행**
   ```bash
   npx tsx scripts/check-admin.ts
   ```

2. **인증 확인 API 테스트**
   ```javascript
   fetch('/api/admin/auth-check', { credentials: 'include' })
     .then(r => r.json())
     .then(data => console.log('인증 상태:', data));
   ```

3. **Vercel 배포 후 로그 확인**
   - Vercel 대시보드 → Deployments → Logs
   - Prisma 클라이언트 관련 오류 확인
