## 🔄 재배포 방법

### 방법 1: 자동 재배포 (일반적)

환경 변수를 추가하면 **자동으로 재배포**가 시작됩니다.

#### 확인 방법

1. **Vercel 대시보드 상단에서 "Deployments" 탭 클릭**
   - 또는 왼쪽 사이드바에서 "Deployments" 클릭

2. **최신 배포 확인**
   - 가장 위에 있는 배포가 최신 배포입니다
   - 상태를 확인:
     - **"Building"** (노란색): 배포 중
     - **"Ready"** (초록색): 배포 완료
     - **"Error"** (빨간색): 배포 실패

3. **배포 완료 대기**
   - 보통 1-3분 정도 소요
   - "Ready" 상태가 되면 배포 완료

---

### 방법 2: 수동 재배포 (필요한 경우)

자동 재배포가 되지 않았거나, 수동으로 재배포하고 싶은 경우:

#### 단계별 방법

1. **Vercel 대시보드에서 "Deployments" 탭 클릭**
   - 상단 메뉴 또는 왼쪽 사이드바

2. **최신 배포 찾기**
   - 가장 위에 있는 배포 확인

3. **재배포 메뉴 열기**
   - 배포 카드 오른쪽 상단에 **⋯** (세 점) 메뉴 아이콘 클릭
   - 또는 배포 카드를 클릭하여 상세 페이지로 이동

4. **"Redeploy" 선택**
   - 드롭다운 메뉴에서 "Redeploy" 또는 "재배포" 선택
   - 확인 메시지가 나타나면 "Redeploy" 클릭

5. **배포 진행 확인**
   - 배포 상태가 "Building"으로 변경됨
   - 완료되면 "Ready" 상태로 변경됨

## 📊 배포 상태 확인

### 배포 상태 아이콘

- **🟡 Building**: 배포 중
  - "Building..." 또는 "Building your project..."
  - 몇 분 정도 소요

- **🟢 Ready**: 배포 완료
  - "Ready" 또는 "Deployment ready"
  - 사이트가 정상적으로 작동함

- **🔴 Error**: 배포 실패
  - "Build Failed" 또는 "Deployment failed"
  - 에러 로그 확인 필요

- **⚪ Queued**: 대기 중
  - "Queued" 또는 "Waiting to build..."
  - 다른 배포가 완료되면 시작됨

## 🔍 배포 로그 확인

### 배포 실패 시 로그 확인

1. **Deployments 탭에서 실패한 배포 클릭**
   - 빨간색 "Error" 상태인 배포 클릭

2. **"Build Logs" 또는 "Functions Logs" 탭 확인**
   - 왼쪽 탭에서 "Logs" 선택

3. **에러 메시지 확인**
   - 빨간색으로 표시된 에러 메시지 확인
   - 환경 변수 관련 에러가 있는지 확인

### 일반적인 배포 시간

## 💡 팁

### 자동 재배포가 안 되는 경우

1. **환경 변수 저장 확인**
   - Settings > Environment Variables에서 저장 완료 확인

2. **수동 재배포 실행**
   - Deployments > 최신 배포 > ⋯ > Redeploy

3. **Git 푸시로 재배포**
   - 로컬에서 작은 변경사항 커밋 후 푸시
   - 자동으로 재배포됨

### 배포 상태 알림 받기

- **이메일 알림**: Vercel 설정에서 활성화 가능
- **Slack 알림**: 통합 설정에서 활성화 가능

## 🚨 문제 해결

### 배포가 실패하는 경우

1. **에러 로그 확인**
   - Deployments > 실패한 배포 > Logs

2. **환경 변수 확인**
   - Settings > Environment Variables
   - 모든 필수 환경 변수가 올바르게 설정되었는지 확인

3. **빌드 에러 확인**
   - TypeScript 에러
   - 의존성 문제
   - 환경 변수 누락
