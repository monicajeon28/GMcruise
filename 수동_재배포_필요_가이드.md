# 수동 재배포 필요 가이드

> **작성일**: 2025년 1월 28일  
> **상황**: 환경 변수 추가 후 자동 재배포가 시작되지 않음

---

## 🔍 현재 상황

- ✅ 환경 변수는 저장되었음
- ❌ 새로운 배포가 생성되지 않음
- ❌ 오늘 날짜 배포가 없음

**해결**: 수동 재배포 필요

---

## 🔄 수동 재배포 방법

### 방법 1: Deployment Details 페이지에서 재배포

1. **현재 Deployment Details 페이지에서**
   - 오른쪽 상단에 **"Visit"** 버튼 옆에 **"..."** (세 점) 메뉴 아이콘 클릭

2. **"Redeploy" 선택**
   - 드롭다운 메뉴에서 "Redeploy" 또는 "재배포" 선택

3. **재배포 모달 확인**
   - "Redeploy" 모달이 나타남
   - Environment: "Production" 선택 확인
   - "Use existing Build Cache" 체크 해제 (권장)

4. **"Redeploy" 버튼 클릭**
   - 모달 하단의 "Redeploy" 버튼 클릭

5. **새 배포 생성 확인**
   - Deployments 탭으로 이동
   - 가장 위에 새 배포가 생성되는지 확인
   - "Building" 상태로 시작됨

---

### 방법 2: Deployments 탭에서 재배포

1. **Deployments 탭 이동**
   - 상단 메뉴에서 "Deployments" 클릭

2. **최신 배포 찾기**
   - 가장 위에 있는 배포 (13일 전 배포)

3. **재배포 메뉴 열기**
   - 배포 카드 오른쪽 상단에 **⋯** (세 점) 메뉴 아이콘 클릭

4. **"Redeploy" 선택**
   - 드롭다운 메뉴에서 "Redeploy" 선택

5. **재배포 모달 확인**
   - "Redeploy" 모달이 나타남
   - Environment: "Production" 선택 확인

6. **"Redeploy" 버튼 클릭**

---

## ⏱️ 재배포 진행 확인

### 배포 상태 확인

1. **Deployments 탭에서 새 배포 확인**
   - 가장 위에 새 배포가 생성됨
   - 상태: **"Building"** (노란색)

2. **배포 완료 대기**
   - 보통 1-3분 소요
   - 상태가 **"Ready"** (초록색)로 변경되면 완료

3. **배포 완료 확인**
   - "Ready" 상태 확인
   - 배포 시간 확인 (오늘 날짜/시간)

---

## 🧪 재배포 완료 후 테스트

### 1. 웹사이트 접속

1. **https://www.cruisedot.co.kr 접속**

2. **강력 새로고침**
   - Windows/Linux: `Ctrl + Shift + R` 또는 `Ctrl + F5`
   - Mac: `Cmd + Shift + R`

3. **로딩 문제 해결 확인**
   - ✅ "Shorts를 불러오는 중..." → 유튜브 영상 표시
   - ✅ "영상을 불러오는 중..." → 유튜브 영상 표시
   - ✅ "상품을 불러오는 중..." → 상품 목록 표시
   - ✅ "커뮤니티 게시글을 불러오는 중..." → 게시글 표시

## 💡 중요 사항

### 환경 변수는 이미 저장되어 있음

- ✅ 환경 변수는 Vercel에 저장되어 있습니다
- ✅ 재배포 시 환경 변수가 자동으로 적용됩니다
- ✅ 별도로 환경 변수를 다시 추가할 필요 없습니다

### 재배포 후 환경 변수 적용

- 재배포가 완료되면 환경 변수가 적용됩니다
- `NEXT_PUBLIC_BASE_URL`, `YOUTUBE_API_KEY`, `DATABASE_URL` 모두 적용됨

## 🚨 문제가 계속되는 경우

### Build Logs 확인

1. **새 배포의 Build Logs 확인**
   - 새 배포 클릭
   - "Build Logs" 섹션 확인
   - 에러가 있는지 확인

2. **에러 해결**
   - 에러 내용에 따라 해결 방법 결정

### 환경 변수 재확인

1. **Settings > Environment Variables 확인**
   - 환경 변수가 올바르게 저장되어 있는지 확인
   - Environment가 "Production" 또는 "All"인지 확인
