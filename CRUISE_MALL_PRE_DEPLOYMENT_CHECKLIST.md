## 📝 점검 방법

### 자동 점검
```bash
# 빌드 확인
npm run build  # ✅ 완료

# 타입 체크
npm run typecheck  # .next/types 에러 있음 (실제 소스 코드 문제 없음)

# 린터 확인
npm run lint  # ✅ 에러 없음
```

### 수동 점검

#### 1. 메인 페이지 플로우
1. 메인 페이지 접속 (`/`)
2. 히어로 섹션 확인
3. "라이브방송 참여" 버튼 클릭 → 스크롤 동작 확인
4. "상품 둘러보기" 버튼 클릭 → 스크롤 동작 확인
5. 상품 목록 확인 (1줄당 5개, 최대 3줄)
6. "전체 보기" 버튼 클릭 → `/product` 새창 열기 확인
7. 커뮤니티 섹션 게시글 클릭 → 로그인 모달 없이 바로 이동 확인

#### 2. 내정보 페이지 플로우
1. 로그인 후 내정보 페이지 접속 (`/my-info` 또는 `/community/my-info`)
2. 사용자 정보 섹션 확인
3. "수정" 버튼 클릭
4. 이름 입력 필드 확인
5. 연락처 입력 필드 확인
6. 정보 입력 후 "저장" 클릭
7. 저장 후 정보 정상 표시 확인

#### 3. 커뮤니티 플로우
1. 비로그인 상태에서 메인 페이지 접속
2. 커뮤니티 섹션 게시글 클릭 → 로그인 모달 없이 상세 페이지 이동 확인
3. 게시글 상세 페이지에서 "글쓰기" 버튼 클릭 → 로그인 페이지로 이동 확인

#### 4. 반응형 확인
1. 모바일 화면 (375px)에서 메인 페이지 확인
2. 태블릿 화면 (768px)에서 메인 페이지 확인
3. 데스크톱 화면 (1920px)에서 메인 페이지 확인
4. 각 화면 크기에서 버튼 클릭 가능 여부 확인

#### 5. API 엔드포인트 테스트
```bash
# 상품 목록 API
curl http://localhost:3000/api/public/products?limit=12

# 커뮤니티 게시글 API
curl http://localhost:3000/api/community/posts?limit=6

# 내정보 API (로그인 필요)
curl -b cookies.txt http://localhost:3000/api/community/my-info
```
