# GitHub 저장소 연결 확인 가이드

> **작성일**: 2025년 1월 28일  
> **에러**: "The provided GitHub repository can't be found."

---

## ⚠️ 에러 메시지

**빨간 글씨**: "The provided GitHub repository can't be found."

**의미**: Vercel이 GitHub 저장소를 찾을 수 없습니다.

---

## 🔍 원인 확인

### 가능한 원인

1. **GitHub 저장소 이름 변경**
   - 저장소 이름이 변경되었을 수 있음

2. **GitHub 저장소 삭제**
   - 저장소가 삭제되었을 수 있음

3. **권한 문제**
   - Vercel이 GitHub 저장소에 접근할 수 없음

4. **연결 해제**
   - Vercel과 GitHub 간 연결이 끊어짐

---

## ✅ 해결 방법

### 옵션 1: 재배포 계속 진행 (빠른 해결)

**중요**: 이 에러는 **재배포를 막지 않습니다**.

환경 변수만 업데이트하고 싶다면:

1. **"Redeploy" 버튼 클릭**
   - 에러 메시지가 있어도 재배포는 가능합니다
   - 환경 변수는 이미 저장되어 있으므로 재배포 시 적용됩니다

2. **배포 완료 확인**
   - Deployments 탭에서 배포 상태 확인
   - "Ready" 상태가 되면 완료

---

### 옵션 2: GitHub 저장소 연결 확인 (근본 해결)

재배포 후에도 문제가 있거나, 자동 배포를 위해 필요:

#### 1단계: Vercel에서 Git 설정 확인

1. **Vercel 대시보드 > Settings > Git 클릭**
   - 왼쪽 사이드바에서 "Git" 클릭
   - 또는 상단 메뉴에서 "Settings" > "Git"

2. **연결된 저장소 확인**
   - GitHub 저장소가 올바르게 연결되어 있는지 확인
   - 저장소 이름 확인
   - 예: `monicajeon28/GMcruise` 또는 `monicajeon28/cruise-guide`

3. **저장소가 없거나 잘못된 경우**
   - "Disconnect" 버튼 클릭
   - 또는 "Connect Git Repository" 클릭

---

#### 2단계: GitHub 저장소 확인

1. **GitHub 접속**
   - https://github.com 접속
   - 로그인

2. **저장소 확인**
   - 저장소가 존재하는지 확인
   - 저장소 이름 확인
   - 예: `GMcruise` 또는 `cruise-guide`

3. **저장소 권한 확인**
   - 저장소에 접근 권한이 있는지 확인
   - Private 저장소인 경우 권한 필요

---

#### 3단계: Vercel과 GitHub 재연결

1. **Vercel > Settings > Git 이동**

2. **"Connect Git Repository" 클릭**
   - 또는 "Disconnect" 후 다시 연결

3. **GitHub 저장소 선택**
   - GitHub 계정 선택
   - 저장소 선택
   - 예: `monicajeon28/GMcruise`

4. **연결 완료 확인**
   - 저장소가 연결되었는지 확인
