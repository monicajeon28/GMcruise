# 링크 관리 시스템 정리 요약

## 현재 링크 구조

### 1. 파트너몰 링크 (필수 - 자동 추적됨)
- **URL**: `/${mallUserId}/shop`
- **예시**: `/boss1/shop`
- **추적 방식**: `AffiliateTracker` 컴포넌트가 쿠키에 `affiliate_mall_user_id`와 `affiliate_code` 저장
- **상태**: ✅ 작동 중
- **용도**: 파트너의 기본 판매몰 페이지

### 2. 랜딩 페이지 (선택적 - 구현 필요)
- **URL**: `/store/${affiliateCode}/${landingSlug}`
- **예시**: `/store/AFF-BOSS1-BECF/boss1`
- **추적 방식**: 동일한 `AffiliateTracker` 사용 가능
- **상태**: ⚠️ 페이지 미구현 (링크만 표시됨)
- **용도**: 파트너 전용 랜딩 페이지 (별도 디자인/컨텐츠)

### 3. 특정 상품 링크 (관리자 생성)
- **URL**: `/products/${productCode}?affiliate=${affiliateCode}&agent=${agentCode}&link=${linkCode}`
- **추적 방식**: 쿼리 파라미터로 추적
- **상태**: ✅ 작동 중
- **용도**: 특정 상품에 대한 마케팅 캠페인 링크

## 어필리에이트 추적에 필요한 링크

### 필수 링크
1. **파트너몰 링크** (`/${mallUserId}/shop`)
   - 모든 파트너가 자동으로 가짐
   - `AffiliateTracker`로 자동 추적
   - 고객이 이 링크로 접속하면 자동으로 해당 파트너에게 할당

### 선택적 링크
2. **랜딩 페이지** (`/store/${affiliateCode}/${landingSlug}`)
   - 파트너가 `affiliateCode`와 `landingSlug`를 설정한 경우만 표시
   - 현재 페이지가 구현되지 않음 (404 발생 가능)
   - 구현 필요 또는 제거 권장

3. **특정 상품 링크**
   - 관리자가 생성한 링크만 표시
   - 쿼리 파라미터로 추적

## 정리 완료 사항

### ✅ 중복 제거
- ~~파트너몰 기본 링크~~ (제거)
- ~~파트너몰 추적 링크~~ (제거)
- **파트너몰 링크** (통합, 자동 추적 표시)

### ✅ UI 개선
- "자동 추적" 배지 추가
- 링크 설명 명확화
- 랜딩 페이지는 있는 경우만 표시

### ⚠️ 확인 필요
- 랜딩 페이지 (`/store/${affiliateCode}/${landingSlug}`) 실제 구현 여부
- 링크 유효성 검사 (CORS 제한으로 정확한 검사 어려움)

## 권장 사항

1. **랜딩 페이지 구현 또는 제거**
   - 구현: `/app/store/[affiliateCode]/[landingSlug]/page.tsx` 생성
   - 제거: 랜딩 페이지 링크 표시 제거

2. **링크 유효성 검사**
   - 클라이언트에서 간단한 검사만 수행 (CORS 제한)
   - 서버 사이드에서 실제 링크 검증 API 추가 고려

3. **링크 통계**
   - 각 링크별 클릭 수, 리드 수, 판매 수 표시 (이미 구현됨)
