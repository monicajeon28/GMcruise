# 🔍 서버 로그 확인 가이드 (초보자용)

## 방법 1: 브라우저 개발자 도구에서 확인 (가장 쉬움) ⭐

### 단계별 설명:

1. **F12 키 누르기**
   - 브라우저에서 F12 키를 누르면 개발자 도구가 열립니다

2. **Network 탭 클릭**
   - 개발자 도구 상단에 있는 "Network" 탭을 클릭합니다

3. **페이지 새로고침**
   - F5 키를 누르거나 새로고침 버튼을 클릭합니다

4. **필터에 "products" 입력**
   - Network 탭 상단의 필터 입력창에 `products`를 입력합니다

5. **`/api/public/products` 클릭**
   - 목록에서 `/api/public/products` 항목을 찾아 클릭합니다

6. **Response 탭 확인**
   - 오른쪽 패널에서 "Response" 탭을 클릭합니다
   - 여기에 서버에서 받은 데이터가 표시됩니다

### 확인할 내용:
- `"ok": true` 인지 확인
- `"products"` 배열에 상품이 있는지 확인
- `"products": []` (빈 배열)이면 상품이 없다는 뜻입니다

---

## 방법 2: 서버 터미널에서 확인

### 서버가 실행 중인 터미널 창 찾기:

1. **터미널 창 찾기**
   - VSCode나 터미널 프로그램에서 `npm run dev` 또는 `yarn dev`를 실행한 창을 찾습니다
   - 보통 검은색 배경에 흰색 글씨로 로그가 계속 출력되는 창입니다

2. **로그 확인**
   - 터미널에서 `[Public Products API]`로 시작하는 로그를 찾습니다
   - 스크롤을 위로 올려서 최근 로그를 확인합니다

### 확인할 로그:
```
[Public Products API] Active AffiliateProducts: {
  count: 1,
  productCodes: ['2026-0510-MSC-01'],
  ...
}

[Public Products API] Found CruiseProducts: {
  count: 1,
  productCodes: ['2026-0510-MSC-01'],
  ...
}
```

---

## 방법 3: 스크린샷으로 공유하기

1. **F12 → Network 탭**
2. **페이지 새로고침 (F5)**
3. **`/api/public/products` 클릭**
4. **Response 탭의 내용을 스크린샷**
5. **스크린샷을 공유**

이렇게 하면 제가 직접 확인할 수 있습니다!

---

## 문제 해결 체크리스트

### 상품이 안 나올 때 확인할 것:

1. ✅ **어필리에이트 수당 관리 페이지에서:**
   - 상품이 등록되어 있는지 확인
   - "노출 상태"가 "노출중"인지 확인
   - "상품 코드"를 메모해두기 (예: `2026-0510-MSC-01`)

2. ✅ **크루즈 상품 관리 페이지에서:**
   - 같은 "상품 코드"로 상품이 등록되어 있는지 확인
   - 예: 어필리에이트 수당에 `2026-0510-MSC-01`이 있으면
   - 크루즈 상품 관리에도 `2026-0510-MSC-01`이 있어야 합니다

3. ✅ **브라우저 콘솔 확인:**
   - F12 → Console 탭
   - 빨간색 에러 메시지가 있는지 확인
