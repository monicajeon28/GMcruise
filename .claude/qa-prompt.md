# QA 테스트 프롬프트

Claude에게 QA를 요청할 때 이 템플릿을 사용하세요.

---

## 사용 방법

```
@qa-prompt.md 이 템플릿대로 개발서버에서 QA 진행해줘
```

또는

```
개발서버 https://cruise-guide-dev.vercel.app 에서 QA 테스트 진행해줘.
배포_및_QA_가이드.md의 체크리스트 참고해서 모든 항목 테스트하고 결과 보고해줘.
```

---

## QA 요청 템플릿

```markdown
# QA 테스트 요청

다음 환경에서 QA 테스트를 수행해주세요:

**환경**: [개발서버 / 운영서버]
**URL**: https://cruise-guide-dev.vercel.app
**브랜치**: dev
**날짜**: 2025-11-18
**변경사항**: [새로운 기능 또는 버그 수정 내용]

## QA 체크리스트

아래 항목들을 순서대로 테스트하고, 각 항목의 결과를 보고해주세요:

### 🔐 1. 인증 시스템
- [ ] 일반 사용자 로그인 (`/login`)
  - 테스트 계정: test@example.com / 1101
- [ ] 관리자 로그인 (`/admin/login`)
  - 관리자 계정으로 로그인
- [ ] 로그아웃 기능
- [ ] 회원가입 (`/signup`)

### 🤖 2. 크루즈가이드 지니 (AI 챗봇)
- [ ] 챗봇 접속 (`/chat`)
- [ ] AI 대화 가능 여부
- [ ] 음성 인식 버튼 작동 (`/translator`)
- [ ] 사진 번역 기능
- [ ] 네비게이션 기능
- [ ] 채팅 히스토리 저장 (새로고침 후 복원)

### 🏪 3. 크루즈몰
- [ ] 상품 목록 로드 (`/products`)
- [ ] 상품 상세 페이지 (`/products/[code]`)
- [ ] 상품 검색 기능
- [ ] 판매원 페이지 (`/[mallUserId]`)

### 👔 4. 어필리에이트
- [ ] 계약 신청 (`/affiliate/contract`)
- [ ] 내 계약 정보 (`/affiliate/my-contract`)
- [ ] 판매 상품 (`/affiliate/products`)
- [ ] 팀 관리 (`/affiliate/team`)

### ⚙️ 5. 관리자 패널
- [ ] 대시보드 접근 (`/admin/dashboard`)
- [ ] 고객 관리 (`/admin/customers`)
- [ ] 상품 관리 (`/admin/products`)
- [ ] 메시지 발송 (`/admin/messages`)
- [ ] 통계 조회 (`/admin/analytics`)

### 📱 6. 모바일 반응형
- [ ] iPhone 뷰 (375px)
- [ ] Android 뷰 (360px)
- [ ] 태블릿 뷰 (768px)
- [ ] 가로/세로 모드 전환

### 🔔 7. 푸시 알림
- [ ] 푸시 알림 구독 가능
- [ ] 알림 수신 테스트
- [ ] 알림 클릭 시 페이지 이동

### 🎨 8. UI/UX
- [ ] 로딩 스피너 표시
- [ ] 에러 메시지 표시 (의도적 에러 발생 시)
- [ ] 빈 상태 UI (데이터 없을 때)
- [ ] 이미지 로딩 및 최적화

### ⚡ 9. 성능
- [ ] 페이지 로드 시간 (< 3초)
- [ ] API 응답 시간 (< 1초)
- [ ] 이미지 최적화 확인

### 🐛 10. 콘솔 에러
- [ ] 브라우저 콘솔에 에러 없음
- [ ] 네트워크 탭에서 실패한 요청 없음
- [ ] 404 에러 없음

## 테스트 방법

1. **브라우저 개발자 도구 열기**
   - F12 또는 Cmd+Option+I

2. **Console 탭 확인**
   - 에러 메시지 체크

3. **Network 탭 확인**
   - 실패한 요청 체크 (빨간색)
   - 느린 요청 체크 (> 1초)

4. **각 기능 실행**
   - 실제 사용자처럼 클릭, 입력, 네비게이션

5. **결과 기록**
   - 스크린샷 (에러 발생 시)
   - 에러 메시지 복사

## 보고 형식

각 항목에 대해 다음 형식으로 보고해주세요:

```
✅ 통과: 일반 사용자 로그인
❌ 실패: 상품 상세 페이지
   - 에러: TypeError: Cannot read property 'name' of undefined
   - 위치: /products/PROD001
   - 콘솔: [복사된 에러 메시지]
   - 스크린샷: [있으면]

⚠️  경고: 채팅 히스토리
   - 작동은 하지만 로딩이 느림 (3초)
   - 개선 권장
```

## 우선순위

- 🔴 Critical: 기본 기능 작동 안 함
- 🟡 High: 일부 기능 오류
- 🟢 Medium: 성능 이슈
- ⚪ Low: UI/UX 개선 사항

## QA 완료 기준

- [ ] 모든 Critical 항목 통과
- [ ] 80% 이상의 High 항목 통과
- [ ] 콘솔에 Critical 에러 없음
- [ ] 주요 사용자 플로우 정상 작동

## 실전 예시

### 예시 1: 새 기능 배포 후 QA

```
@qa-prompt.md

개발서버에서 "결제 시스템" 기능 QA 해줘.

특히 다음 항목 중점 확인:
- 결제 페이지 로드
- 결제 정보 입력
- 결제 완료 플로우
- 에러 핸들링
```

### 예시 2: 버그 수정 후 QA

```
@qa-prompt.md

"채팅 히스토리 저장" 버그 수정했어.
개발서버에서 다음 테스트해줘:

1. /chat 접속
2. 몇 개 메시지 보내기
3. 새로고침
4. 메시지 복원되는지 확인
5. localStorage 대신 DB에서 불러오는지 확인
```

```

운영 서버 배포 전 전체 QA 진행해줘.

상세 보고서 작성해줘.

환경: https://cruise-guide-dev.vercel.app
