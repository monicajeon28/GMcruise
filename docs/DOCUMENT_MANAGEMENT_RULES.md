# 서류관리 시스템 활용 룰

> **작성일**: 2025-01-28  
> **목적**: 타사 비교 견적서, 구매확인서, 환불완료증서 생성 및 활용 가이드  
> **대상**: 관리자, 대리점장, 판매원

---

## 📋 개요

서류관리 시스템은 어필리에이트 판매 과정에서 필요한 3가지 문서를 생성하고 관리하는 기능입니다.

### 지원 문서 타입

1. **타사 비교 견적서** - 고객에게 여러 상품 옵션을 비교하여 제공
2. **구매확인서** - 구매 완료 후 고객에게 발송하는 확인서
3. **환불완료증서** - 환불 처리가 완료된 고객에게 발송하는 증서

---

## 🔐 권한 체계

### 관리자 (Admin)
- ✅ **모든 권한**: 모든 판매에 대한 문서 생성 가능
- ✅ **전체 조회**: 모든 판매 내역 조회 및 문서 생성
- ✅ **감사 추적**: 모든 문서 생성 이력 확인 가능

### 대리점장 (Manager)
- ✅ **자신의 판매**: 본인이 담당한 판매에 대한 문서 생성 가능
- ✅ **하위 판매원 판매**: 본인 팀의 판매원이 담당한 판매에 대한 문서 생성 가능
- ⚠️ **제한**: 다른 대리점장의 판매에는 접근 불가

### 판매원 (Agent)
- ✅ **자신의 판매**: 본인이 직접 담당한 판매에 대한 문서 생성 가능
- ⚠️ **제한**: 다른 판매원의 판매에는 접근 불가

---

## 📄 문서별 활용 가이드

### 1. 타사 비교 견적서

#### 사용 시점
- 고객이 여러 업체의 가격을 비교하고 싶어할 때
- 경쟁사 대비 우리 상품의 가격 경쟁력을 보여주고 싶을 때
- 고객이 구매 결정을 망설일 때

#### 생성 조건
- 판매 상태가 `PENDING`, `CONFIRMED`, `PAID`인 경우
- 환불되거나 취소된 판매는 생성 불가

#### 필수 입력 정보
- 고객명 (필수)
- 상품코드 (필수)
- 우리 가격 (필수)
- 타사 가격 비교 정보 (1개 이상)

#### 선택 입력 정보
- 고객 전화번호
- 고객 이메일
- 상품명
- 인원수
- 객실타입
- 요금카테고리

#### 발송 방법
- SMS/카카오톡: 고객 전화번호가 있는 경우 자동 발송
- 이메일: 고객 이메일이 있는 경우 자동 발송
- 수동 다운로드: 관리자 패널에서 PDF 다운로드 가능 (추후 구현)

#### 활용 팁
- 타사 가격은 정확한 정보를 입력하세요
- 비고란에 각 업체의 특징이나 차이점을 명시하면 좋습니다
- 고객에게 이메일로 발송하면 더 상세한 비교표를 확인할 수 있습니다

---

### 2. 구매확인서

#### 사용 시점
- 결제가 완료된 직후
- 고객이 구매 확인을 요청할 때
- 구매 내역을 공식적으로 전달해야 할 때

#### 생성 조건
- 판매 상태가 `CONFIRMED` 또는 `PAID`인 경우
- 자동 발송: 결제 웹훅에서 자동으로 발송됨
- 수동 생성: 관리자 패널에서 수동으로 재발송 가능

#### 자동 발송 정보
- 판매 완료 시 담당자(판매원 또는 대리점장)가 설정되어 있으면 자동 발송
- SMS, 카카오톡, 이메일 중 설정된 채널로 발송

#### 수동 생성 시
- 관리자 패널에서 판매 내역 선택 후 "구매확인서" 버튼 클릭
- 즉시 고객에게 발송됨

#### 포함 정보
- 고객명
- 상품명 및 상품코드
- 결제금액
- 구매일자
- 주문번호
- 인원수, 객실타입 등 상세 정보
- 담당자 정보

---

### 3. 환불완료증서

#### 사용 시점
- 환불 처리가 완료된 후
- 고객이 환불 확인을 요청할 때
- 환불 내역을 공식적으로 전달해야 할 때

#### 생성 조건
- 판매 상태가 `REFUNDED`인 경우만 생성 가능
- 환불 처리가 완료된 후에만 생성 가능

#### 필수 정보
- 원래 결제금액
- 환불금액
- 환불일자
- 환불사유
- 주문번호

#### 발송 방법
- SMS/카카오톡: 고객 전화번호가 있는 경우 자동 발송
- 이메일: 고객 이메일이 있는 경우 자동 발송
- 수동 다운로드: 관리자 패널에서 PDF 다운로드 가능 (추후 구현)

#### 활용 팁
- 환불사유는 명확하게 기록하세요
- 환불금 입금 안내를 포함하여 고객 불안을 해소하세요
- 환불 완료 후 즉시 발송하는 것이 좋습니다

## 🚀 대시보드 배포 계획

### Phase 1: 관리자 패널 (현재)
- ✅ 관리자만 접근 가능
- ✅ 모든 판매 내역 조회 및 문서 생성
- ✅ 위치: `/admin/affiliate/documents`

### Phase 2: 대리점장 대시보드 (예정)
- 📍 위치: `/affiliate/dashboard` 또는 `/partner/dashboard`
- 📋 기능:
  - 본인 담당 판매 목록 조회
  - 본인 팀 판매원의 판매 목록 조회
  - 문서 생성 및 발송
  - 발송 이력 확인

### Phase 3: 판매원 대시보드 (예정)
- 📍 위치: `/affiliate/agent/dashboard`
- 📋 기능:
  - 본인 담당 판매 목록만 조회
  - 문서 생성 및 발송
  - 발송 이력 확인

## 📊 문서 생성 이력 관리

### 자동 기록
- 모든 문서 생성 시 `AffiliateInteraction` 테이블에 기록
- 기록 정보:
  - 문서 타입
  - 생성 일시
  - 생성자 정보
  - 판매 ID
  - 리드 ID

### 이력 조회
- 관리자: 모든 문서 생성 이력 조회 가능
- 대리점장: 본인 및 팀 판매원의 문서 생성 이력 조회 가능
- 판매원: 본인의 문서 생성 이력만 조회 가능

## ⚠️ 주의사항

### 데이터 정확성
- 타사 비교 견적서의 경쟁사 가격은 정확한 정보를 입력하세요
- 허위 정보 입력 시 법적 문제가 발생할 수 있습니다

### 개인정보 보호
- 고객 정보는 안전하게 관리하세요
- 문서 발송 시 개인정보가 포함되므로 주의하세요

### 발송 타이밍
- 구매확인서는 결제 완료 후 즉시 발송하는 것이 좋습니다
- 환불완료증서는 환불 처리 완료 후 즉시 발송하세요
- 타사 비교 견적서는 고객이 구매 결정을 내리기 전에 발송하세요

### 권한 준수
- 본인의 권한 범위를 벗어난 판매에 대한 문서는 생성할 수 없습니다
- 권한이 없는 판매에 접근하려고 하면 오류가 발생합니다
